# 第五阶段、Backend 后端系统详解

**前置知识**：建议先学习第三、四阶段内容，了解 DeepAgents 的文件系统中间件和基本使用方法。

## 一、Backend 概述

### 1.1 什么是 Backend？

Backend 是 DeepAgents 框架的核心组件之一，它为智能体提供了**环境交互能力**。简单来说，Backend 决定了 Agent 在哪里工作、如何存储文件、如何执行命令。

**核心作用**：
- **存储引擎**：管理文件的读写和持久化
- **执行环境**：提供安全的代码执行沙箱
- **上下文扩展**：通过外部存储解决上下文窗口限制
- **环境隔离**：保护宿主机安全，防止危险操作

### 1.2 Backend 与 FilesystemMiddleware 的关系

```
┌─────────────────────────────────────┐
│      FilesystemMiddleware           │
│  (管理文件操作的逻辑层)              │
└──────────────┬──────────────────────┘
               │ 调用
               ▼
┌─────────────────────────────────────┐
│           Backend                   │
│  (实际存储和执行的底层实现)          │
└─────────────────────────────────────┘
```

- **FilesystemMiddleware**：负责处理文件操作的逻辑（如路径解析、权限检查、大结果自动转存）
- **Backend**：负责实际的存储和执行（如写入磁盘、在 Docker 中执行命令）

### 1.3 五种 Backend 类型

| Backend 类型 | 说明 | 适用场景 | 安全性 |
|-------------|------|---------|--------|
| **FilesystemBackend** | 直接操作本地磁盘 | 本地开发、文档生成、文件管理 | ⭐⭐⭐ (依赖 virtual_mode) |
| **DockerBackend** | 在 Docker 容器中执行 | 代码执行、依赖安装、危险操作 | ⭐⭐⭐⭐⭐ (完全隔离) |
| **StoreBackend** | 存储在数据库中 | 知识检索、长期记忆、跨会话共享 | ⭐⭐⭐⭐ (数据库权限控制) |
| **E2BBackend** | 使用 E2B 云端沙箱 | 云端执行、多租户、生产环境 | ⭐⭐⭐⭐⭐ (云端隔离) |
| **CompositeBackend** | 混合模式 | 复杂场景、多后端协作 | ⭐⭐⭐⭐⭐ (灵活组合) |

## 二、FilesystemBackend - 本地文件系统

### 2.1 功能说明

**FilesystemBackend** 是最常用的 Backend 类型，它将 Agent 的文件操作映射到本地磁盘。

**核心特性**：
- 支持虚拟文件系统模式（`virtual_mode=True`）
- 自动将绝对路径（如 `/workspace/file.md`）映射到本地目录
- 大工具结果自动转存（>2KB 自动写入文件）
- 支持分页读取（默认每次 500 行）

### 2.2 基础使用示例

```python
from deepagents import create_deep_agent
from deepagents.backends import FilesystemBackend
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

# 1. 加载环境变量
load_dotenv(override=True)

# 2. 初始化模型 - 使用 MiniMax-M2.5
model = ChatOpenAI(
    model="MiniMax-M2.5",
    api_key=os.getenv("MINIMAX_API_KEY"),
    base_url="https://api.minimaxi.com/v1",
    temperature=0
)

# 3. 配置 FilesystemBackend
# root_dir: 本地根目录
# virtual_mode=True: 启用虚拟文件系统，Agent 使用绝对路径映射到 root_dir
backend = FilesystemBackend(
    root_dir="./workspace",
    virtual_mode=True
)

# 4. 创建 Agent
agent = create_deep_agent(
    model=model,
    backend=backend,
    system_prompt="你是一个文件系统操作助手。"
)

# 5. 调用 Agent
config = {"configurable": {"thread_id": "fs_demo"}}

# 写入文件
result = agent.invoke({
    "messages": [{"role": "user", "content": "创建一个名为 hello.py 的文件，内容是 print('Hello World')"}]
}, config=config)

# 读取文件
result = agent.invoke({
    "messages": [{"role": "user", "content": "读取 hello.py 文件内容"}]
}, config=config)

# 列出目录
result = agent.invoke({
    "messages": [{"role": "user", "content": "列出当前目录的文件"}]
}, config=config)
```

### 2.3 虚拟文件系统模式

**virtual_mode** 是 FilesystemBackend 的重要参数：

| 模式 | 说明 | 示例 |
|------|------|------|
| `virtual_mode=True` | Agent 使用绝对路径（如 `/file.md`），自动映射到 `root_dir` | Agent 写入 `/test.md` → 实际保存到 `./workspace/test.md` |
| `virtual_mode=False` | Agent 直接操作本地文件系统，路径不加限制 | Agent 可以访问系统任意文件（危险！） |

**推荐配置**：始终设置 `virtual_mode=True` 以保证安全。

### 2.4 分页读取配置

FilesystemBackend 默认每次读取 500 行，可以通过系统提示词自定义：

```python
system_prompt = """你是一个文件处理助手。

在使用 read_file 工具时，请遵循以下规则：
1. 对于大型文件（超过 300 行），每次只读取 300 行
2. 使用 offset 参数进行分页读取
3. 示例：read_file(file_path="/large.txt", offset=0, limit=300)
"""

backend = FilesystemBackend(root_dir="./workspace", virtual_mode=True)

agent = create_deep_agent(
    model=model,
    backend=backend,
    system_prompt=system_prompt
)
```

### 2.6 优缺点分析

**优点**：
- ✅ 简单易用，无需额外配置
- ✅ 直接操作本地文件，调试方便
- ✅ 支持虚拟文件系统，安全可靠
- ✅ 自动大结果转存，避免上下文溢出

**缺点**：
- ❌ 不支持代码执行（无沙箱）
- ❌ 无法跨会话共享（除非使用外部存储）
- ❌ 性能受限于本地磁盘 I/O

**适用场景**：
- 本地文件管理
- 文档生成和编辑
- 代码编写（不执行）
- 数据分析和报告生成

## 三、DockerBackend - Docker 容器沙箱

### 3.1 功能说明

**DockerBackend** 为 Agent 提供了一个**完全隔离的 Docker 容器环境**，所有文件操作和命令执行都在容器中进行。

**核心优势**：
- **安全隔离**：Agent 的命令不会影响到宿主机
- **环境可控**：可以指定 Python 版本、预装依赖
- **支持 execute**：可以安全执行 Shell 命令
- **自动清理**：任务完成后容器自动销毁

### 3.2 工作原理

```
┌─────────────────────────────────────┐
│         宿主机 (你的电脑)            │
│                                     │
│  ┌───────────────────────────────┐  │
│  │    Docker Container           │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │    Agent 工作环境       │  │  │
│  │  │  - /workspace           │  │  │
│  │  │  - Python 3.11          │  │  │
│  │  │  - 预装依赖             │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
│                                     │
└─────────────────────────────────────┘
```

### 3.3 基础使用示例

```python
from deepagents import create_deep_agent
from deepagents.backends import DockerBackend
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv
import os

load_dotenv(override=True)

# 1. 初始化模型 - 使用 MiniMax-M2.5
model = ChatOpenAI(
    model="MiniMax-M2.5",
    api_key=os.getenv("MINIMAX_API_KEY"),
    base_url="https://api.minimaxi.com/v1",
    temperature=0
)

# 2. 配置 DockerBackend
backend = DockerBackend(
    image="python:3.11",  # 使用 Python 3.11 镜像
    volumes={"./workspace": "/workspace"}  # 挂载本地目录到容器
)

# 3. 创建 Agent
agent = create_deep_agent(
    model=model,
    backend=backend,
    system_prompt="""你是一个在 Docker 容器中工作的编程助手。
    
你可以：
1. 编写和修改代码
2. 执行 Python 脚本
3. 安装依赖包
4. 运行测试

所有操作都在隔离的 Docker 容器中进行，不会影响宿主机。
"""
)

# 4. 执行任务
config = {"configurable": {"thread_id": "docker_demo"}}

# 任务：创建并执行 Python 脚本
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": """请完成以下任务：
1. 在 /workspace 目录下创建一个计算阶乘的 Python 文件 factorial.py
2. 执行该文件，计算 10 的阶乘
3. 返回执行结果"""
    }]
}, config=config)

print(result["messages"][-1].content)

# 5. 清理资源
backend.close()  # 关闭并销毁容器
```

from deepagents import create_deep_agent
from deepagents.backends import DockerBackend
from langchain_openai import ChatOpenAI

async def run_docker_demo():

        )

1. 创建一个 Python 脚本 data_analysis.py，要求：
   - 生成 100 个随机数（正态分布，均值=50，标准差=10）

        print("\n📋 执行任务...")
        result = agent.invoke({
            "messages": [{"role": "user", "content": task}]
        }, config=config)

            for f in os.listdir(workspace_dir):
                print(f"  - {f}")

## 四、StoreBackend - 数据库存储

### 4.1 功能说明

**StoreBackend** 将文件存储在数据库中（如 Redis、Postgres），支持**跨会话共享**和**高效检索**。

**核心特性**：
- 持久化存储，跨会话保留
- 支持语义搜索和检索
- 适合长期记忆和知识库
- 需要配合 `store` 参数使用

### 4.2 基础使用示例

```python
from deepagents import create_deep_agent
from deepagents.backends import StoreBackend
from langgraph.store.memory import InMemoryStore
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv
import os

load_dotenv(override=True)

# 1. 初始化模型 - 使用 MiniMax-M2.5
model = ChatOpenAI(
    model="MiniMax-M2.5",
    api_key=os.getenv("MINIMAX_API_KEY"),
    base_url="https://api.minimaxi.com/v1",
    temperature=0
)

# 2. 配置 Store
# InMemoryStore 是内存中的临时存储，生产环境可使用 Redis 或 Postgres
store = InMemoryStore()

# 3. 配置 StoreBackend
backend = StoreBackend(store=store)

# 4. 创建 Agent（需要传入 store 参数）
agent = create_deep_agent(
    model=model,
    backend=backend,
    store=store,  # 必须传入 store
    system_prompt="""你是一个知识管理助手。
    
你可以：
1. 将重要知识存储到数据库中
2. 检索历史知识
3. 跨会话共享信息
"""
)

# 5. 执行任务
config = {"configurable": {"thread_id": "store_demo"}}

# 存储知识
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "请记住：Python 的列表推导式语法是 [expression for item in iterable if condition]"
    }]
}, config=config)

# 检索知识（新会话）
config2 = {"configurable": {"thread_id": "store_demo_session2"}}
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "我之前让你记住的 Python 列表推导式是什么？"
    }]
}, config=config2)

print(result["messages"][-1].content)
```

## 五、E2BBackend - 云端沙箱

### 5.1 功能说明

**E2BBackend** 使用 [E2B](https://e2b.dev) 提供的**云端沙箱服务**，在云端执行代码和命令。

**核心优势**：
- 云端执行，无需本地资源
- 多租户隔离，适合生产环境
- 支持多种编程语言
- 按需付费，成本可控

### 5.2 安装依赖

```bash
pip install e2b
```

### 5.3 完整使用示例

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from langchain_core.messages import BaseMessage, ToolMessage
from dotenv import load_dotenv
import os
import sys

# 1. 导入 E2B Backend
try:
    from e2b import Sandbox
    from deepagents.backends.sandbox import BaseSandbox
    from deepagents.backends.protocol import ExecuteResponse, SandboxBackendProtocol
except ImportError:
    print("请先安装 e2b: pip install e2b")
    sys.exit(1)

load_dotenv(override=True)

# 2. 实现 E2BBackend
class E2BBackend(BaseSandbox):
    """E2B 沙箱后端实现"""
    
    def __init__(self, template: str = "base", api_key: str = None):
        self.sandbox = Sandbox(template=template, api_key=api_key)
        super().__init__()
    
    @property
    def id(self) -> str:
        return self.sandbox.sandbox_id
    
    def execute(self, command: str) -> ExecuteResponse:
        """执行命令"""
        execution = self.sandbox.commands.run(command)
        return ExecuteResponse(
            output=execution.output or "",
            error=execution.error or ""
        )
    
    def write_file(self, path: str, content: str) -> None:
        """写入文件"""
        self.sandbox.files.write(path, content)
    
    def read_file(self, path: str) -> str:
        """读取文件"""
        return self.sandbox.files.read(path)
    
    def close(self) -> None:
        """关闭沙箱"""
        self.sandbox.close()

# 3. 运行演示
async def run_e2b_demo():
    print("\n" + "="*80)
    print("DeepAgents E2BBackend 云端沙箱演示")
    print("="*80)
    
    # 1. 检查 API Key
    e2b_api_key = os.getenv("E2B_API_KEY")
    if not e2b_api_key:
        print("❌ 未找到 E2B_API_KEY，请在 .env 文件中配置")
        return
    
    # 2. 初始化模型 - 使用 MiniMax-M2.5
    llm = ChatOpenAI(
        model="MiniMax-M2.5",
        api_key=os.getenv("MINIMAX_API_KEY"),
        base_url="https://api.minimaxi.com/v1",
        temperature=0
    )
    
    # 3. 初始化 E2B Backend
    print("☁️ 正在连接 E2B 云端沙箱...")
    try:
        backend = E2BBackend(template="base", api_key=e2b_api_key)
        print(f"✅ 沙箱已启动 (ID: {backend.id})")
    except Exception as e:
        print(f"❌ 沙箱启动失败：{e}")
        return
    
    try:
        # 4. 创建 Agent
        agent = create_deep_agent(
            model=llm,
            backend=backend,
            system_prompt="""你是一个在 E2B 云端沙箱中工作的编程助手。
            
你可以：
1. 编写和运行代码
2. 安装依赖包
3. 执行 Shell 命令
4. 将结果保存到云端文件系统
"""
        )
        
        # 5. 执行任务
        config = {"configurable": {"thread_id": "e2b_demo"}}
        
        task = """请在云端沙箱中完成以下任务：

1. 创建一个 Python 脚本，计算斐波那契数列的前 20 项
2. 执行该脚本
3. 返回结果"""

        print("\n📋 执行任务...")
        result = agent.invoke({
            "messages": [{"role": "user", "content": task}]
        }, config=config)
        
        print("\n" + "="*80)
        print("☁️ 执行结果")
        print("="*80)
        print(result["messages"][-1].content)
        
    finally:
        # 6. 清理资源
        print("\n🧹 正在关闭云端沙箱...")
        backend.close()
        print("✅ 沙箱已关闭")
        print("演示结束")

# 运行演示
import asyncio
asyncio.run(run_e2b_demo())
```

### 5.4 优缺点分析

**优点**：
- ✅ 云端执行，不占用本地资源
- ✅ 多租户完全隔离
- ✅ 支持多种语言
- ✅ 生产级安全性

**缺点**：
- ❌ 需要网络连接
- ❌ 按使用量付费
- ❌ 依赖第三方服务

**适用场景**：
- 生产环境部署
- 多租户 SaaS 应用
- 需要弹性计算资源
- 代码执行即服务（Code as a Service）

## 六、CompositeBackend - 混合模式

### 6.1 功能说明

**CompositeBackend** 允许**组合多个 Backend**，发挥各自优势。

**典型场景**：
- 本地存文件 + Docker 跑代码
- StoreBackend 存知识 + DockerBackend 执行

### 6.2 实现示例

```python
from deepagents import create_deep_agent
from deepagents.backends import FilesystemBackend, DockerBackend
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv
import os

load_dotenv(override=True)

class CompositeBackend:
    """混合 Backend：本地存文件 + Docker 执行"""
    
    def __init__(self, fs_backend: FilesystemBackend, docker_backend: DockerBackend):
        self.fs_backend = fs_backend  # 本地文件系统
        self.docker_backend = docker_backend  # Docker 执行
    
    def write_file(self, path: str, content: str):
        """写入文件到本地"""
        return self.fs_backend.write_file(path, content)
    
    def read_file(self, path: str) -> str:
        """从本地读取文件"""
        return self.fs_backend.read_file(path)
    
    def execute(self, command: str):
        """在 Docker 中执行命令"""
        return self.docker_backend.execute(command)
    
    def close(self):
        """清理资源"""
        self.docker_backend.close()

# 使用示例
fs_backend = FilesystemBackend(root_dir="./workspace", virtual_mode=True)
docker_backend = DockerBackend(image="python:3.11")

composite = CompositeBackend(fs_backend, docker_backend)

# 初始化模型 - 使用 MiniMax-M2.5
model = ChatOpenAI(
    model="MiniMax-M2.5",
    api_key=os.getenv("MINIMAX_API_KEY"),
    base_url="https://api.minimaxi.com/v1",
    temperature=0
)

agent = create_deep_agent(
    model=model,
    backend=composite,
    system_prompt="""你是一个混合后端工作的助手。
    
- 文件操作使用本地文件系统（快速、持久化）
- 代码执行使用 Docker 容器（安全、隔离）
"""
)

# 执行任务
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": """1. 在本地创建一个 Python 文件 analysis.py
2. 在 Docker 容器中执行该文件
3. 将结果保存到本地"""
    }]
})

# 清理
composite.close()
```

### 6.3 优缺点分析

**优点**：
- ✅ 灵活组合，发挥各自优势
- ✅ 性能优化（本地存储 + 云端执行）
- ✅ 成本优化（按需使用不同后端）

**缺点**：
- ❌ 实现复杂度高
- ❌ 需要管理多个后端

**适用场景**：
- 复杂企业应用
- 需要性能和隔离兼顾
- 混合云部署

### 7.2 选择建议

```
┌─────────────────────────────────────┐
│  你的需求是什么？                    │
└──────────────┬──────────────────────┘
               │
    ┌──────────┼──────────┐
    │          │          │
    ▼          ▼          ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│本地开发 │ │代码执行 │ │生产部署 │
└────┬────┘ └────┬────┘ └────┬────┘
     │           │           │
     ▼           ▼           ▼
Filesystem   Docker     E2BBackend
Backend      Backend    + StoreBackend
+ virtual_mode
```

**决策树**：

1. **只需要文件读写？**
   - 是 → **FilesystemBackend**（简单、快速）
   - 否 → 继续

2. **需要执行代码/命令？**
   - 是 → **DockerBackend**（本地安全）或 **E2BBackend**（云端）
   - 否 → 继续

3. **需要跨会话共享？**
   - 是 → **StoreBackend**（Redis/Postgres）
   - 否 → **FilesystemBackend**

4. **生产环境多租户？**
   - 是 → **E2BBackend** 或 **CompositeBackend**
   - 否 → **DockerBackend**

### 7.3 最佳实践

### 9.1 核心价值
