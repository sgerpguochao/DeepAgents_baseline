## 二、核心入口：create_deep_agent()

这是整个框架的核心函数，它创建了一个功能完整的深度智能体。

**源码位置**：`D:\sorfware_install\python3.8_install\envs\deepagent_1\Lib\site-packages\deepagents\graph.py`

### 2.1 默认配置

- **默认模型**：使用 Claude Sonnet 4 或 GPT-4o 作为默认模型（推荐）
- **集成工具**：默认集成 7 个核心文件操作工具
- **待办事项**：提供待办事项管理功能
- **子代理支持**：支持子代理调用

### 2.2 关键参数

| 参数 | 说明 |
|------|------|
| model | 支持自定义语言模型 |
| tools | 自定义工具集 |
| system_prompt | 系统提示词 |
| subagents | 子代理配置 |
| backend | 文件存储后端 |
| interrupt_on | 人机交互配置 (Human-in-the-Loop)。允许在特定节点暂停 Agent 执行，等待人工干预。 |

### 2.3 基础使用示例

```python
from deepagents import create_deep_agent
from langchain_tavily import TavilySearch
from langgraph.checkpoint.memory import InMemorySaver
from dotenv import load_dotenv
from langchain_deepseek import ChatDeepSeek

# 1. 加载 .env 环境变量
load_dotenv(override=True)

# 2. 初始化模型
model = ChatDeepSeek(model="deepseek-chat", temperature=0)

# 3. 初始化 Tavily 搜索工具
tavily = TavilySearch(max_results=3)

# 4. 编写系统提示词
research_instructions = """
您是一位资深的研究人员...
"""

# 5. 创建 DeepAgents 智能体
agent = create_deep_agent(
    name="DeepAgents_Agent",
    tools=[tavily],
    model=model,
    system_prompt=research_instructions,
    checkpointer=InMemorySaver(),
)

# 6. 配置线程 ID
config = {"configurable": {"thread_id": "1"}}

# 7. 调用智能体
result = agent.invoke(
    {"messages": [{"role": "user", "content": "查询内容"}]},
    config=config
)

# 8. 获取结果
print(result["messages"][-1].content)
```

## 四、提示词系统

DeepAgents 内置了多个提示词组件，用于控制智能体的行为。

### 4.1 文件系统提示词

**文件路径**：`deepagents/middleware/filesystem.py`

**核心功能**：
- 定义文件系统操作的安全规则
- 规定文件路径的处理方式
- 提供文件操作的最佳实践指导

### 4.2 执行工具提示词

**文件路径**：`deepagents/middleware/filesystem.py`

**核心功能**：
- 定义工具调用的参数规范
- 说明工具返回值的处理方式
- 提供错误处理指导

### 4.3 待办事项提示词

**文件路径**：`deepagents/middleware/todo.py`

**核心功能**：
- 定义任务分解的格式（Markdown）
- 说明任务状态的含义（Pending/Completed）
- 提供任务管理的最佳实践

### 4.4 子代理提示词

**文件路径**：`deepagents/middleware/subagent.py`

**核心功能**：
- 定义子代理调用协议
- 规定 task 工具的参数结构
- 说明结果回传方式
- 设置安全底线

## 五、规划工具系统 (Planning System / Todo List)

### 5.1 核心组件

- **组件**：TodoListMiddleware
- **工具名**：write_todos

### 5.2 功能说明

Agent 在行动前先生成 Markdown 格式的待办事项列表（Todo List），并在执行过程中更新状态（完成/进行中）。

### 5.3 工作流程

1. **任务接收**：Agent 接收复杂任务
2. **任务分解**：调用 write_todos 将任务拆解为子步骤
3. **状态更新**：每完成一步，更新状态为 Completed
4. **自我反思**：在每一步行动前，Agent 都会看到当前的 Todo List

### 5.4 待办事项状态

| 状态 | 说明 |
|------|------|
| Pending | 待执行 |
| in_progress | 进行中 |
| Completed | 已完成 |

## 七、总结

### 7.1 DeepAgents 核心价值

- **开箱即用**：集成了文件系统操作、任务规划、子代理管理等核心功能

- 深度研究任务
- 复杂任务规划
- 自动化工作流
- 企业级应用

DeepAgents 作为 LangChain 生态系统中的"代理工具"：

- **LangChain**：框架
- **LangGraph**：运行时
- **DeepAgents**：代理工具

三者形成完整的技术栈，为 AI 应用开发提供全方位支持。

## 九、create_deep_agent 的 Skills 支持【补充说明】

DeepAgents 框架通过 `create_deep_agent` 函数提供强大的 Skills 功能支持，允许用户加载自定义的技能文件来扩展 Agent 的能力。

### 9.1 Skills 概述

Skills 是 DeepAgents 框架的一个重要特性，它允许用户：

- 定义自定义行为规范和指令
- 创建可重用的技能模板
- 在多个 Agent 之间共享技能配置

### 9.2 skills 参数说明

| 参数 | 说明 |
|------|------|
| skills | 可选参数，技能源路径列表（如 `["/skills/user/", "/skills/project/"]`） |
| 类型 | `list[str] \| None` |
| 默认值 | `None` |
| 加载方式 | 使用 StateBackend 时通过 `invoke(files={...})` 传递；使用 FilesystemBackend 时从磁盘加载 |

### 9.3 使用示例

**基本用法**：

```python
from deepagents.graph import create_deep_agent

# 创建带有 skills 的 deep agent
agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-5-20250929",
    skills=["/skills/user/", "/skills/project/"],  # 指定 skill 路径
    tools=[...],  # 可选：自定义工具
    subagents=[...],  # 可选：子代理配置
)

# 调用 agent
result = agent.invoke({"messages": [{"role": "user", "content": "你的任务"}]})
```

**子代理也支持 Skills**：

```python
# 在子代理配置中也可以指定 skills
agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-5-20250929",
    subagents=[
        {
            "name": "researcher",
            "description": "进行深入研究",
            "prompt": "你是一位研究员...",
            "skills": ["/skills/research/"],  # 子代理专属 skills
            "tools": [...],
        }
    ]
)
```

### 9.4 注意事项

- 路径必须使用 POSIX 风格（正斜杠 `/`）
- 路径是相对于 backend 的 root 目录
- 相同名称的 skill，后加载的会覆盖先加载的（last one wins）
- Skills 通过 SkillsMiddleware 添加到 Agent 的中间件栈中

### 9.5 源码解析

在 `graph.py` 中，Skills 的实现逻辑如下：

```python
# 1. 主 Agent 的 skills 支持 (第 272-273 行)
if skills is not None:
    deepagent_middleware.append(SkillsMiddleware(backend=backend, sources=skills))

# 2. 通用子代理的 skills 支持 (第 210-211 行)
if skills is not None:
    gp_middleware.append(SkillsMiddleware(backend=backend, sources=skills))

# 3. 用户自定义子代理的 skills 支持 (第 250-252 行)
```
